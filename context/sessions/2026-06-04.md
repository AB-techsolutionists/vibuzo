## Key Decisions
- **NL first, explicit fallback** — Agent always tries rigid syntax match first; if it doesn't match, falls through to natural language inference. Backward compatible by design.
- **Type inference heuristics** — `/add-context` maps keywords to types: "always/never/convention" → pattern, "must/required/naming" → standard, "flows/architecture/decision" → architecture
- **Name generation** — Agent extracts 2-4 key nouns from statement, kebab-cases them, checks uniqueness, confirms with user before creating
- **Single clarifying question** — Never present a form. If ambiguous, ask ONE question at a time.
- **Conversation history** — Agent checks last 3 messages to resolve ambiguous types or missing content
- **Approval gates left untouched** — approval-gates.md and related commands not modified (out of scope per spec)
