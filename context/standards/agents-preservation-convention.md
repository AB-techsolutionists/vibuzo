# AGENTS.md Preservation Convention

Standard for preserving user customizations to AGENTS.md across Vibuzo installer updates.

## The Preservation Boundary

AGENTS.md contains a marker at the bottom that defines the update safety boundary:

```
─── PASTE YOUR CUSTOM RULES BELOW THIS LINE ───
```

Everything **below** this marker is preserved across installer updates. Everything **above** (the framework sections, Karpathy Principles, core rules) is overwritten.

## Three Scenarios

When the installer runs, it checks the existing AGENTS.md and handles three cases:

### Case 1: Fresh Install (no AGENTS.md)
Download the latest AGENTS.md from the repository. No preservation needed.

### Case 2: Vibuzo file with custom rules (marker detected)
1. Save everything below the `─── PASTE YOUR CUSTOM RULES BELOW THIS LINE ───` marker
2. Download the fresh AGENTS.md from the repository
3. Re-append the saved custom rules below the marker in the new file

### Case 3: User-owned AGENTS.md (exists but no marker)
The user has their own AGENTS.md (not generated by Vibuzo). In this case:
1. Save the entire existing content
2. Download the fresh AGENTS.md from the repository
3. Prepend the user's content above Vibuzo's content, separated by `---`

```
<User's original AGENTS.md content>

---

<Vibuzo's AGENTS.md content>

─── PASTE YOUR CUSTOM RULES BELOW THIS LINE ───
```

The user's content always appears first, with a `---` separator before Vibuzo's content. The user can then move sections between the two as they see fit.

## Approval Gate

Before touching AGENTS.md, the installer must:
1. Detect which of the 3 cases applies
2. Explain what will happen in a clear description box
3. Ask `Proceed? (y/N)` and wait for user confirmation
4. Only proceed if the user responds "y" or "yes"

This ensures the user is never surprised by AGENTS.md being modified.

## Implementation

Both installers implement this logic:
- `install.ps1` — PowerShell implementation with `-ForegroundColor` output
- `install.sh` — Bash implementation with ANSI escape codes

Search for `# AGENTS.md` section in each installer for the implementation.
