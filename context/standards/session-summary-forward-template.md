# Session Summary Forward Template

The session summary file (`context/sessions/YYYY-MM-DD-<title>.md`) uses a dual-layer design: forward-looking agent sections above, and an auto-generated styled box (Session Compaction) below. Both are generated by the agent during `/session`.

## Core Rule

**The forward sections and the compaction box serve different purposes — no duplication.**

| Feature | Forward Sections (above) | Session Compaction Box (below) |
|---|---|---|
| Purpose | Curated, forward-looking intent | Structured overview of what happened |
| Viewpoint | What matters NEXT session | What WAS this session |
| Progress | Done / In Progress / Blocked (categorized) | Progress (done, in-progress, blocked) |
| Decisions | Forward Decisions (curated table) | Key Decisions (all significant) |
| Files | Hot Files (max 8, why hot next session) | Relevant Files (all key files touched) |
| Context | Forward Context (what's actionable next) | Critical Context (git state, gotchas) |
| Constraints | Constraints & Preferences (shaped decisions) | Constraints & Preferences (all) |
| Goal | (implicit in Session Summary) | Goal (explicit one-liner) |
| Next Steps | (not included — in compaction box) | Next Steps (actionable steps) |

## Template Structure

```
## Session Summary           ← 3-5 sentence changelog. Facts only.
## Constraints & Preferences ← Format rules, conventions that governed the session.
## Progress                  ← Done / In Progress / Blocked. Reference specific files.
## Forward Decisions         ← Curated decisions that shape NEXT session. Table format.
## Forward Context           ← State, gotchas, unfinished work next session MUST know. 2-5 items.
## Next Steps                ← Standard workflow instructions.
## Hot Files                 ← Max 8 files likely touched next session. Curated, not exhaustive.
## Timeline Entry            ← Single row for the master index.
## Session Compaction        ← Auto-generated styled box (Goal, Constraints, Progress, Key Decisions, Next Steps, Critical Context, Relevant Files).
```

## Section Details

### Session Summary
A tight 3-5 sentence paragraph telling the complete story. Changelog-style — specific, factual, no fluff. Example:
> "Added version tracking (0.x.x semver) to Vibuzo, removed the source-mirror sync convention, and saved the large-document-size-gate pattern. Three commits pushed."

### Constraints & Preferences
Rules and preferences that governed the session's decisions. Bullet list with bold topics. Only include what actually shaped decisions — don't list every preference the user has ever stated.

### Progress
Three subsections: Done, In Progress, Blocked. Done items reference specific files and features. Use *(none)* for empty sections.

### Forward Decisions
Table format with #, Decision, and Rationale columns. Only include decisions that will matter in the next session. If a decision was made but won't affect future work, omit it.

### Forward Context
2-5 bullet points that the next session MUST know. Focus on:
- Unfinished work and where to pick up
- Gotchas and hard-won lessons
- State that compaction won't capture (e.g., "this pattern isn't wired into agent rules yet")
- Relationships between changed files

### Next Steps
No longer needed in forward sections — compaction box auto-generates Next Steps. Forward sections focus on Forward Context, Forward Decisions, and Hot Files.

### Hot Files
Table with File and Why Hot columns. Curated to 5-8 files most likely to be edited or read next session. Not every file touched this session. The "Why Hot" column should be specific — e.g., "4 strings to bump — highest miss risk" not just "needs update."

## Forward vs Compaction Distinction

Even though both are generated by the agent, they serve different roles:

| Forward Section | Compaction Box Section | Why Different |
|---|---|---|
| **Forward Decisions** (curated table, max ~5) | Key Decisions (all significant decisions) | Forward curates only what shapes NEXT session; compaction captures everything worth recording |
| **Forward Context** (actionable gotchas) | Critical Context (git state + raw gotchas) | Forward synthesizes "what must I know to continue?"; compaction reports raw state |
| **Hot Files** (max 8, why hot next session) | Relevant Files (all key files with relevance) | Forward targets next likely edits; compaction inventories everything important |

## Origin

Established in `versioning-mirror-cleanup` (2026-06-07) when the 11-section template was consolidated to 7 forward-looking sections alongside a compaction paste target. Updated in `session-auto-compaction` (2026-06-08) when `/session` began auto-generating the compaction block — the paste target became an auto-generated styled box, and forward sections shifted focus to purely forward-looking content.
